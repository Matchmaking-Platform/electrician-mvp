# 10 分钟演示脚本

照着这份脚本走一遍,完整覆盖 MVP 全部 P0 功能。**所有账号密码统一 `password123`**。

> 准备:确保 `pnpm dev` 在跑 + 数据库已起。打开两个浏览器窗口(或一个常规 + 一个隐身),分别登顾客和电工。

---

## 0:30 — 开场:首页 + 公开浏览(无需登录)

1. 打开 [http://localhost:3000](http://localhost:3000)
2. **要讲**:三个入口(找电工 / 登录 / 注册)
3. 点 **"先随便看看"** → 跳到 `/electricians`
4. **要讲**:
   - 左边筛选器(关键词、服务区、最低评分、价格区间、在线开关)
   - 右上排序(综合 / 评分 / 价格 ↑↓)+ 网格/列表切换
   - URL 同步:改任意筛选,地址栏跟着变,刷新保持
5. 点任意一位 → 详情页
6. **要讲**:头部信息 + 案例 / 报价 / 评价 三 tab。"立即下单"未登录会跳 `/login`。

## 2:00 — 顾客发布订单(展示地图选址)

1. 用 `customer1@example.com` 登录
2. 顾客仪表盘 → **"发布新订单"**
3. 5 步表单逐步走:
   - **服务类型**:维修(默认即可)
   - **描述**:标题 `卫生间灯不亮` + 详细描述(随便写)+ 可选上传 1-2 张问题图
   - **地址 + 地图**:填地址 `上海市浦东新区张江路 88 号`,**在地图上点一下**录经纬度
   - **时间 + 价格**:时间留空(表示尽快),预算 `200`
   - **确认**:看到所有信息汇总,点"确认发布"
4. **要讲**:跳到订单详情,状态 = 待接单。

## 3:00 — 电工抢单(展示实时大厅)

1. 切到第二个浏览器,用 `electrician1@example.com`(陈师傅,浦东)登录
2. 电工工作台 → **接单大厅**
3. **要讲**:刚才那单已出现(地址含"浦东"匹配陈师傅服务区)
4. 同时演示:用 `electrician3@example.com`(王师傅,虹口)在第三个隐身窗登,他**看不到**这单 → 派单匹配生效
5. 回到陈师傅那边,**点"抢单"**
6. **要讲**:跳到电工侧订单详情,状态变 已接单。如果别人也点 → 看到 409 "已被抢"

## 4:30 — 实时聊天(展示 Socket.io)

1. 双方都在订单详情页 → **聊天面板自动连接**(右上角"已连接"+"对方在线")
2. **顾客侧**输入"你好师傅,我家在 3 楼,电梯坏了" → 发送
3. **电工侧**立即收到(<1 秒)
4. **电工侧**回复"好的,我带工具,预计 1 小时到"
5. 演示**敏感词屏蔽**:顾客发"我手机 13800001234" → 电工看到的是 `我手机 [屏蔽]`
6. 演示**图片消息**:任何一方点📷 → 选张图 → 对方实时看到

## 5:30 — 支付托管(Mock)

1. 顾客侧订单详情 → **"立即支付"**
2. 对话框默认填 ¥200(预算)→ 确认
3. **要讲**:状态徽章变 "已托管"。MVP 阶段是 Mock(无 Stripe);代码里留好 Stripe 接口,生产换 key 即可
4. **同时:顾客铃铛会在 ~30 秒内亮红点**(此前其它通知:接单)→ 不用等,继续

## 6:30 — 电工完工 + 顾客确认

1. **电工侧**订单详情 → "上传完工凭证" → 选一张图 → 提交
2. 状态变 **待确认完工**
3. **顾客侧**刷新页(或等通知)→ 看到 "待确认完工"
4. **顾客侧** → "确认完工"(浏览器 confirm 提示 → 确定)
5. **自动弹出评价对话框** —— 4 个维度都点 5 星 + 评论 + 可选图片 → 提交
6. **要讲**:订单完成 + 已结算

## 7:30 — 电工钱包 + 评价生效

1. 电工侧 → **钱包**
2. **要讲**:可用余额 = 顾客付的金额 - 平台抽成(MVP 抽成 0,等于全到电工)
3. 演示**申请提现**:点"申请提现"→ 填银行信息 + 金额 → 提交
4. 切到 `/electrician/reviews` → 看到刚才的 5 星评价
5. **要讲**:电工可以回复一次

## 8:30 — 管理员后台(KPI + 图表)

1. 用 `admin@example.com` 在第三个窗口登录
2. 进 `/admin` → **要讲**:
   - 4 张 KPI 卡(GMV / 平台抽成 / 完成订单 / 进行中)
   - 2 张图(7 天订单/注册曲线 + GMV 柱状)
   - 7 个功能入口
3. **提现审批**:进 "提现审批" → 找到电工刚才的申请 → "标记已打款"
4. **要讲**:电工的可用余额相应扣减,操作日志记录
5. **资质审核 / 评价审核 / 用户管理 / 操作日志** 都点一下展示

## 9:30 — 安全 + 测试

1. 回终端展示:
   - `pnpm test` → 94 单测全过 (~200ms)
   - `pnpm test:e2e` → 6 个 E2E 全过 (~25s),覆盖**所有主流程**
2. **要讲**:架构亮点
   - 抢单原子性:Prisma `updateMany WHERE status=PENDING`,后到 409
   - 资金一致性:确认完工时用 transaction 同时更新订单 + 电工余额
   - Socket 鉴权:cookie 透传 NextAuth JWT,无需重新登录
   - 派单匹配:address ⊇ serviceArea 子串(可换 PostGIS)
   - 评分重算:hide/unhide 评价后自动重算 avgRating

## 10:00 — 收尾

> "MVP 已经覆盖了双边市场完整闭环。代码里所有 Mock 部分(Stripe / 邮件 / 上传)都留了正式接口位置,生产部署见 DEPLOYMENT.md。"

## 如果只有 5 分钟的迷你版

跳过聊天细节、敏感词、管理后台,核心走:
1. 公开列表 → 选电工 (30s)
2. 顾客下单(地图)→ 电工抢单(60s)
3. 支付 → 完工 → 确认(90s)
4. 评价 → 钱包(60s)
5. 管理后台一瞥(60s)
